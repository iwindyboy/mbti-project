# 사주팔자 앱 - 코드 검토 & Cursor 개발 가이드

---

## 1. 파일 검토 결과

### ✅ 잘 된 부분

- **입춘 날짜 200년치 수록** — 1900~2100년 SOLAR_TERMS 입춘 날짜, 2017년부터 2월 3일 처리까지 정확함
- **음력 달 대소 구분** — 큰달(30일)/작은달(29일) 구분 데이터 있음
- **윤달 월 표시** — `leap` 필드로 어느 달에 윤달이 끼는지 표시됨
- **DB 콘텐츠 품질** — 오행 5가지 × 운세 6가지 = 30세트, 공감 문체 잘 작성됨
- **오프라인 구조** — API 없이 로컬 파일로 동작하는 설계 자체는 좋음

---

### ❌ 반드시 수정해야 할 버그 (치명적)

#### 버그 1: 윤달 일수 데이터 누락 ← 가장 심각
```js
// 현재 (문제)
1900: { leap: 8, months: [30,29,30,29,30,29,30,30,29,30,29,30] }
//                          ↑ 12개 원소만 있음. 윤8월의 일수가 없음!

// 올바른 형태
1900: { leap: 8, leapDays: 29, months: [30,29,30,29,30,29,30,30,29,30,29,30] }
//               ↑ 윤달의 실제 일수 추가 필요
```
**영향**: 윤달이 있는 해에 음력→양력 변환 계산이 틀림. 1900~2100 중 약 60개 연도에서 날짜 오류 발생.

---

#### 버그 2: 월주(月柱) 계산 불가 — 절기 데이터 부재
현재 SOLAR_TERMS에는 **입춘 1개**만 있음. 월주 계산에는 12개 절기가 필요함.

| 월 | 절기 (이 날부터 해당 월로 전환) |
|----|-------------------------------|
| 寅월 (1월) | 입춘 ✅ 있음 |
| 卯월 (2월) | **경칩** ❌ 없음 |
| 辰월 (3월) | **청명** ❌ 없음 |
| 巳월 (4월) | **입하** ❌ 없음 |
| 午월 (5월) | **망종** ❌ 없음 |
| 未월 (6월) | **소서** ❌ 없음 |
| 申월 (7월) | **입추** ❌ 없음 |
| 酉월 (8월) | **백로** ❌ 없음 |
| 戌월 (9월) | **한로** ❌ 없음 |
| 亥월 (10월) | **입동** ❌ 없음 |
| 子월 (11월) | **대설** ❌ 없음 |
| 丑월 (12월) | **소한** ❌ 없음 |

**영향**: 월주를 계산할 수 없어서 사주팔자 중 월주가 항상 틀림.

---

#### 버그 3: 오행 판별 로직 없음
saju-db.js에는 콘텐츠만 있고, **어떤 기준으로 DB를 참조하는지 로직이 없음**.

현재:
- DB 키: `木`, `火`, `土`, `金`, `水`
- 문제: 사용자의 생년월일로 계산한 사주에서 어떤 기둥(연/월/일/시)의 오행을 기준으로 DB를 참조하는지 정의 안 됨

일반적으로 **일간(日干)의 오행**을 기준으로 결과를 보여주는 게 표준임.

---

### ⚠️ 개선 권장 사항 (선택적)

| 항목 | 현재 상태 | 권장 |
|------|----------|------|
| 입춘 시각 | 날짜만 | 정밀도 위해 시각 추가 고려 (생략 가능) |
| 연주 계산 | 입춘 기준 ✅ | 이미 좋음 |
| 일주 계산 | 율리우스력 공식 ✅ | 이미 좋음 |
| 시주 계산 | 없음 | 간단한 테이블로 추가 필요 |
| export 방식 | `const` 선언만 | HTML script 태그 사용 시 OK |

## 2. Cursor에게 일 시키는 방법 (프롬프트 가이드)

### 📌 Cursor 사용 시 핵심 원칙

1. **항상 기존 파일 먼저 주기** — "saju-calendar.js, saju-db.js 파일을 참고해서" 라고 시작
2. **요청은 1개씩** — 기능 하나씩 나눠서 요청, 한 번에 다 달라고 하면 엉망이 됨
3. **결과물 형식 지정** — "index.html 단일 파일로", "script 태그로 분리해서" 등 명시
4. **검증 기준 알려주기** — "1995년 2월 3일생은 1994년 갑술년이어야 해" 같은 테스트 케이스 제공

---

### 🔧 Step 1: 윤달 데이터 수정 요청

```
saju-calendar.js의 LUNAR_CALENDAR 데이터를 수정해줘.

문제: 현재 각 연도에 `leap` 필드(윤달이 몇 월인지)는 있는데,
      윤달의 실제 일수(29일 or 30일)가 없어.

수정 방법:
- 모든 연도에 `leapDays` 필드 추가
- leap: 0이면 leapDays: 0
- leap: 0이 아닌 연도는 실제 윤달 일수(29 or 30)로 채워줘

기준: 한국천문연구원 음력 데이터
출력: 수정된 LUNAR_CALENDAR 전체 (형식 유지)
```

---

### 🔧 Step 2: 12절기 데이터 추가 요청

```
saju-calendar.js에 월주 계산용 12절기 데이터를 추가해줘.

추가할 상수명: SOLAR_TERMS_MONTHLY
형식:
{
  '경칩': { 2000: '0306', 2001: '0305', ... },
  '청명': { 2000: '0404', ... },
  ... (입춘 포함 12개 절기 모두)
}

- 연도 범위: 1930~2050 (핵심 범위)
- 날짜 형식: 'MMDD' (기존 입춘과 동일)
- 기준: 한국 표준시(KST) 기준 절기 날짜
- 각 절기와 해당하는 월지(月支) 대응표도 주석으로 추가

기존 SOLAR_TERMS 구조와 동일한 방식으로 작성해줘.
```

---

### 🔧 Step 3: 사주 계산 엔진 요청

```
아래 두 파일을 참고해서 사주 계산 함수를 포함한 saju-engine.js를 만들어줘.
- saju-calendar.js (LUNAR_CALENDAR, SOLAR_TERMS 사용)
- saju-db.js (SAJU_CONTENT_DB 사용)

구현할 함수:

1. calculateSaju(year, month, day, hour, isLunar, gender)
   - 입력: 생년월일시, 양음력 여부, 성별
   - 출력: { 연주, 월주, 일주, 시주, 오행, 일간 }

2. getSajuContent(오행)
   - 입력: '木' | '火' | '土' | '金' | '水'
   - 출력: SAJU_CONTENT_DB에서 해당 오행의 6가지 운세 전체

계산 규칙:
- 연주: 입춘(SOLAR_TERMS['입춘'][year]) 기준으로 연간 결정
  → 입춘 이전 출생이면 전년도 간지 사용
- 월주: SOLAR_TERMS_MONTHLY의 절기 기준으로 월간 결정
- 일주: 율리우스력 계산 (기준일: 갑자일 JD=2299160)
  → dayStemIndex = (JD + 40) % 10
  → dayBranchIndex = JD % 12
- 시주: 출생 시간대별 지지 배정 (자시=23~1시, 축시=1~3시...)
- 오행: 일간(日干)의 오행으로 결정
  → 甲乙=木, 丙丁=火, 戊己=土, 庚辛=金, 壬癸=水

음력 입력이면 먼저 양력으로 변환 후 계산.

파일 상단에 10천간, 12지지 배열도 정의해줘.
```

### 🔧 Step 4: index.html UI 요청

```
사주팔자 앱의 index.html을 만들어줘.
아래 파일들을 script 태그로 불러와서 사용해:
- saju-calendar.js
- saju-db.js
- saju-engine.js

=== UI 구조 ===

[입력 화면]
- 이름 입력 (텍스트)
- 생년월일 (연/월/일 드롭다운)
- 태어난 시간 (0~23시, "모름" 옵션 포함)
- 양력/음력 선택 (라디오)
- 성별 선택 (라디오)
- "사주 보기" 버튼

[결과 화면]
- 이름 + "님의 사주팔자" 타이틀
- 사주 8글자 표시 (연주/월주/일주/시주 카드 형태)
- 오행 결과 표시 (예: "🌿 나무(木) 기운")
- 탭 형태로 6가지 운세 표시
  → 재물운 / 애정운 / 직업운 / 성격운 / 건강운 / 가족운
- 각 운세: 메인 콘텐츠 + 실천 팁 4개 + 격려 메시지
- "다시 하기" 버튼

=== 디자인 요건 ===
- 타겟: 20~30대 여성
- 스타일: 따뜻하고 감성적, 한국 트렌드 감성
- 색상: 파스텔 계열, 오행별 색상 적용
  (木=초록, 火=빨강/오렌지, 土=노랑/갈색, 金=흰/회, 水=파랑)
- 폰트: Google Fonts Noto Sans KR
- 반응형 (모바일 우선)
- 외부 라이브러리 없이 vanilla CSS/JS만 사용

=== 기술 요건 ===
- 단일 파일 (index.html)
- script는 body 하단에서 saju-calendar.js → saju-db.js → saju-engine.js 순서로 로드
- 에러 처리: 범위 밖 연도(1930년 이전, 2050년 이후)는 안내 메시지 표시
```

### 🧪 Step 5: 검증 테스트 케이스 요청

```
saju-engine.js의 calculateSaju 함수가 올바르게 동작하는지 
다음 테스트 케이스로 검증하는 코드를 콘솔에서 실행할 수 있게 추가해줘.

테스트 케이스:
1. 1995년 2월 3일 → 연주: 甲戌 (입춘 이전이므로 1994년 기준)
2. 1995년 2월 4일 → 연주: 乙亥 (입춘 당일부터 1995년 기준)
3. 2000년 1월 1일 → 연주: 己卯 (2000년 입춘 이전 → 1999년 기준)
4. 음력 1990년 5월 15일 → 양력 1990년 6월 7일로 변환 확인

console.log로 각 결과를 출력하고, 예상값과 실제값을 비교해줘.
틀린 케이스가 있으면 어디서 계산이 잘못됐는지도 알려줘.
```

## 3. 전체 개발 순서 요약

```
Phase 1: 데이터 수정
  └─ saju-calendar.js에 leapDays 필드 추가 (Step 1)
  └─ 12절기 데이터 SOLAR_TERMS_MONTHLY 추가 (Step 2)

Phase 2: 엔진 구현
  └─ saju-engine.js 작성 (Step 3)
  └─ 테스트 케이스로 검증 (Step 5)

Phase 3: UI 구현
  └─ index.html 작성 (Step 4)
  └─ 디자인 마무리

Phase 4: 배포
  └─ Netlify / GitHub Pages / Vercel
```
