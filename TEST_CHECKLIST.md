# SCAN 성향검사 테스트 체크리스트

## ✅ 완료된 기능
- [x] 설문 진행 (5개 그룹, 각 6문항)
- [x] 답변 선택 및 스크롤
- [x] 이전/다음 버튼 네비게이션
- [x] 결과 계산 로직 (5개 축 점수 합산)
- [x] Grey Zone 감지 (16~20점 범위)
- [x] 결과 페이지 기본 구조
- [x] 베이직 리포트 섹션 (장점/단점/조언)
- [x] 프리미엄 섹션 UI
- [x] 32개 유형 데이터 (scanResultData.ts)

## 🔍 테스트해야 할 항목

### 1. 설문 진행 테스트
- [ ] 모든 30개 문항 답변 가능한지 확인
- [ ] 각 그룹에서 6개 문항 모두 답변해야 다음 단계 버튼 활성화되는지 확인
- [ ] 이전 버튼으로 돌아갔을 때 답변이 유지되는지 확인
- [ ] 스크롤이 부드럽게 작동하는지 확인

### 2. 결과 계산 테스트
- [ ] 브라우저 콘솔에서 typeCode가 제대로 생성되는지 확인
- [ ] 각 축의 점수가 6~30점 범위 내인지 확인
- [ ] Grey Zone (16~20점)이 제대로 감지되는지 확인
- [ ] 역채점(direction: 'minus')이 제대로 작동하는지 확인

### 3. 결과 페이지 테스트
- [ ] typeCode가 화면에 표시되는지 확인
- [ ] 유형 제목이 SCAN_TYPE_DETAILS에서 제대로 로드되는지 확인
- [ ] 장점/단점/조언이 제대로 표시되는지 확인
- [ ] 5개 축 그래프가 제대로 표시되는지 확인
- [ ] Grey Zone 축이 노란색으로 표시되는지 확인
- [ ] 프리미엄 섹션이 제대로 표시되는지 확인

### 4. 데이터 매칭 테스트
- [ ] 모든 32개 유형 코드가 SCAN_TYPE_DETAILS에 있는지 확인
- [ ] 생성된 typeCode가 SCAN_TYPE_DETAILS에 매칭되는지 확인
- [ ] 매칭되지 않는 typeCode가 있을 경우 fallback이 작동하는지 확인

### 5. UI/UX 테스트
- [ ] 모바일 화면에서 레이아웃이 깨지지 않는지 확인
- [ ] 버튼 클릭이 정상 작동하는지 확인
- [ ] 로딩 시간이 적절한지 확인
- [ ] 에러 메시지가 사용자 친화적인지 확인
